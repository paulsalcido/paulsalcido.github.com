My repositories

py-log-tail

This is a python script for tailing logs that rotate, whether they rotate in and out with dates, or if they are one file with a changing inode, this handles both and can even remember where it was if you have to stop and restart it, so it is also useful for processes that tail your logs constantly and push data someplace (as long as you override the default behavior). I have a demo with rabbitmq also in the repository. It also handles files that have been gzipped since your last run, and remembers its place within them as it reads, and will read all the gzipped files in order.

In all honesty, I did this as a project to learn how to work with python. It's my first python code and I decided to do something interesting with it, and the active tailing problem is something that people have to deal with a lot. I actually use it at work for tailing logs on the development boxes when I'm working, because they have date rotations and gzip files, something that this script handles quite well.

You can find the code for this at http://github.com/paulsalcido/py-log-tail/.

pc-webapp

This is a logical starting point for a web application using perl. It has database connectivity, Google OpenID and Facebook Logins built in. Also, the web application can deploy its entire database, and set up its first administrative user itself. It can also track itself using RabbitMQ, Google Analytics, and it has the Twitter Bootstrap library built into it with Template Toolkit as a front end. Perhaps I'll blog about getting a highly available model on a new web server for this application soon.

You can find the code for this at http://github.com/paulsalcido/pc-webapp/.
